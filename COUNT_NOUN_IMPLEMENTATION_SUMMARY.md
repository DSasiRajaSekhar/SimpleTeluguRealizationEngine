Count Noun / Person-Count Implementation Summary
================================================

Purpose
-------

This document closes the controlled count-noun/person-count implementation
step. It records the problem, the representation chosen, the implementation
boundary, and the verified outputs.

Problem
-------

The expression:

```text
reMdu + person-count + prakAsaM plural
```

must surface as:

```text
ixxaru prakAsAlu
```

It must not be generated by treating `reMdu` as an ordinary noun and applying
ordinary plural formation. The earlier ordinary noun path produced the wrong
type of form for this use.

Implemented Representation
--------------------------

Count expressions are represented as an optional `<quantifier>` inside the same
noun phrase that they modify.

Person count:

```xml
<subject type="nominative">
<quantifier type="numeral" counttype="person">
<word pos="numeral">reMdu</word>
</quantifier>
<word pos="noun" gender="masculine" number="plural" person="third" casemarker="">prakAsaM</word>
</subject>
```

Object count:

```xml
<subject type="nominative">
<quantifier type="numeral" counttype="object">
<word pos="numeral">reMdu</word>
</quantifier>
<word pos="noun" gender="nonmasculine" number="plural" person="third" casemarker="">pustakaM</word>
</subject>
```

Implementation Boundary
-----------------------

The implementation is deliberately narrow.

Implemented:

```text
counttype="person"
counttype="object"
special person-count lookup through src/xml/personcount.xml
fallback person-count expression + maMxi
ordinary noun realization for the head noun
```

Not implemented in this step:

```text
time quantifiers
amount quantifiers
measure/unit quantifiers
compound proper noun plural behavior
```

Verified Count Outputs
----------------------

```text
kg_021_ixxaru_prakAsAlu.xml -> ixxaru prakAsAlu
kg_022_mugguru_rAmamurwulu.xml -> mugguru rAmamurwulu
kg_026_reMdu_pustakAlu.xml -> reMdu pustakAlu
kg_027_iravE_Exu_maMxi.xml -> iravE Exu maMxi
```

Regression Result
-----------------

The original 14 evaluation examples remain unchanged.

The Krishnamurti evaluation batch runs successfully. Non-count examples remain
unchanged, while the deliberately updated count examples now use the controlled
`<quantifier>` representation.

Research Note
-------------

This is useful for the paper because it shows an extension that is not a
surface-form shortcut. The input keeps roots and grammatical information, and
the surface forms are produced through a separate count-expression mechanism.

The feature remains separate from ordinary noun plural generation, so forms
such as `reMdu -> ixxaru` are not added to `plural.xml`.
