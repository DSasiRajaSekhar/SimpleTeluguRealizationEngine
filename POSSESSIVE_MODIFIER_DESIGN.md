Possessive Modifier Design
==========================

Purpose
-------

This note defines the next controlled XML and implementation design for
possessive modifiers in Telugu noun phrases.

The immediate motivation comes from Batch 03 of the Krishnamurti examples:

```text
Awanu nA snehiwudu
nA peru rAmArAvu
vAru mA vAru
ive nA puswakAlu
```

These should not be generated by placing surface possessive forms such as `nA`
or `mA` directly as ordinary adjectives.  The input should preserve the source
root and let the existing noun/pronoun morphology produce the possessive form.

Current Code Observation
------------------------

The current `PronounClass` and `NounClass` already contain useful behavior for
genitive/possessive-like forms:

```text
nenu + non-nominative + yoVkka -> nA
nuvvu + non-nominative + yoVkka -> nI
vAdu + non-nominative + yoVkka -> vAdi
vAdu plural + non-nominative + yoVkka -> vAlYlYa
rAmudu + non-nominative + yoVkka -> rAmudi
```

This means the possessive extension should reuse existing morphology rather
than introduce a lexical lookup table.

Controlled XML Shape
--------------------

Add an optional `possessive` element inside an ordinary noun phrase:

```xml
<subject type="nominative">
<possessive type="genitive">
<word pos="pronoun" gender="masculine" number="singular" person="first" casemarker="yoVkka">nenu</word>
</possessive>
<word pos="noun" gender="nonmasculine" number="singular" person="third" casemarker="">kalaM</word>
</subject>
```

Expected output:

```text
nA kalaM
```

For a plural possessor:

```xml
<subject type="nominative">
<possessive type="genitive">
<word pos="pronoun" gender="masculine" number="plural" person="third" casemarker="yoVkka">vAdu</word>
</possessive>
<word pos="noun" gender="nonmasculine" number="singular" person="third" casemarker="">illu</word>
</subject>
```

Expected output:

```text
vAlYlYa illu
```

Design Rule
-----------

For the first implementation:

```text
possessive phrase = possessor in genitive/oblique form + head noun phrase
```

The possessor should be realized as:

```text
internal phrase type: genitive
case marker: yoVkka
```

The existing noun/pronoun case logic already turns `yoVkka` into a blank
surface suffix after deriving the oblique/genitive stem.

Implementation Boundary
-----------------------

The first implementation should support:

```text
pronoun possessors: nenu, memu, nuvvu, mIru, vAdu, vAru, vAlYlYu, etc.
noun/proper noun possessors: rAmudu, subbArAvu, Ayana, etc. where current
                             noun/pronoun morphology can derive the form
one possessive modifier per noun phrase
possessive + ordinary noun head
possessive + plural noun head
possessive + adjective + noun head
```

Do not implement yet:

```text
multiple possessive modifiers in one noun phrase
nested possessives
emphatic clitic -e after possessive phrases
negative equational clauses
semantic ownership beyond the XML-specified possessor
```

Examples to Enable
------------------

After implementation, these Batch 03 examples should become XML candidates:

```text
kg_038 Awanu nA snehiwudu
kg_041 vAru mA vAru
kg_042 nA peru rAmArAvu
```

The example below needs both possessive support and emphatic `-e`, so it should
remain deferred:

```text
kg_048 ive nA puswakAlu
```

Suggested Implementation Steps
------------------------------

```text
1. Add PossessiveElement to store one internal SOCElement possessor.
2. Add optional PossessiveElement field to SOCElement.
3. Add PossessiveElementBuilder to parse <possessive>.
4. Wire SOCElementBuilder to attach the possessive modifier.
5. Add PossessivePhraseBuilder that reuses NounClass/PronounClass.
6. Join possessive phrase before quantifier/adjective/head noun in NounPhraseBuilder.
7. Extend InputXmlValidator to accept exactly one <possessive> in a noun phrase.
8. Add XML for kg_038, kg_041, and kg_042 only after regression passes.
```

Validator Expectations
----------------------

The validator should accept:

```text
<possessive type="genitive"> inside subject/object/complement
exactly one direct <word> child
word pos="noun" or pos="pronoun"
word casemarker="yoVkka"
required gender, number, person attributes
```

The validator should reject:

```text
more than one possessive element in the same phrase
possessive without type="genitive"
possessive word without casemarker="yoVkka"
possessive combined with propernouncompound in the first implementation
surface possessive forms placed as ordinary adjectives to bypass generation
```

Expected No-Regression Behavior
-------------------------------

Existing XML without `possessive` must remain unchanged:

```text
original 14 examples
Krishnamurti noun plural examples
count-noun examples
coupled proper-noun examples
Batch 03 supported simple demonstrative examples
```
